# AGENTS.md

本文件用于固化本项目（当前仓库）的长期协作约束。后续 agent 在本目录工作时默认遵守。

## 1. 项目目标与边界

- 项目是电子书重命名工具，核心脚本：`rename_books_by_meta.py`。
- 支持 `.epub` / `.pdf`，原则是**预览优先**，避免误改名。
- 默认行为应偏保守：宁可不改，也不要错误改名。

## 2. Python 编码习惯

- Python 代码必须带类型标注（尤其是新增函数）。
- 优先使用 `pathlib.Path`，避免裸字符串路径拼接。
- 优先写可测试的纯函数，再由 UI/CLI 组装调用。
- 不引入不必要的重量依赖；标准库优先。
- 小步修改，避免大规模无关重构。
- 变更应与现有风格保持一致（函数命名、注释风格、错误处理风格）。

## 3. 重命名业务约束（关键）

- 仍然采用“预览 -> 用户确认 -> 应用”的流程。
- 支持**选择性重命名**，不是默认全量重命名。
- 以下文件默认不选中重命名：
  - `rfc*.pdf`（例如 `rfc1928.txt.pdf`）
  - 识别出的目标名包含可疑字符（乱码/异常符号等）
- 冲突处理保持安全（同名冲突自动后缀）。
- 任何会提升误改风险的策略调整，必须同步测试与文档。

## 4. 文件索引约束（轻量文本）

- 使用文本索引，不使用 SQLite。
- 索引文件：`file_index.json`（位于应用配置目录）。
- 至少记录：
  - `size`
  - `mtime_ns`
  - `sha256`
  - `updated_at`
- 允许在不可写环境降级（打印 warning，不中断主流程）。

## 5. UI/交互约束

- Qt GUI 与 Textual TUI 都要支持“只对选中项应用重命名”。
- GUI 表格应支持右键动作：
  - 打开文件（系统关联应用）
  - 在文件管理器中定位文件（平台允许时选中）
- 新增交互文案必须同步 `locales/*.json`，不能只改单语言。

## 6. CLI 约束

- `--apply` 仅应用到“选中/过滤后”的条目。
- 过滤参数（如 `--only` / `--exclude`）变更时，必须保持可预览、可解释。
- 预览输出应包含足够信息让用户判断风险（如跳过原因、重复提示）。

## 7. 测试与验证

- 默认测试命令：`python3 -m pytest -q`
- 新增策略必须有单元测试（至少覆盖正常路径 + 边界路径）。
- 涉及索引文件写入的测试，必须用 `tmp_path` + monkeypatch 重定向索引路径，避免污染真实用户目录。

## 8. 文档同步要求

- 行为变化必须同步：
  - `README.md`
  - `docs/USER_GUIDE.md`
  - 必要时 `RENAME_STRATEGY.md`
- 文档要明确“默认跳过规则”和“如何手动选中重命名”。

## 9. 提交规范（如需提交）

- Git 提交信息遵循 Conventional Commits v1.0.0：
  - `type(scope)!: <emoji> <description>`
- emoji 映射按仓库上级约定执行。

## 10. 非目标（避免）

- 不把索引系统升级为数据库（除非用户明确要求）。
- 不在未确认情况下改变默认重命名策略为激进模式。
- 不引入与当前功能无关的架构改造。
